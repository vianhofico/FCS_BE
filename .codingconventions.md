# FCS Backend - Coding Conventions

## Package Naming Rules

### Implementation Packages
- **Rule**: Use `impl` instead of `implements` for service implementation packages
- **Why**: `implements` is a reserved keyword in Java, causing compilation issues
- **Pattern**: `com.fcs.be.<module>.service.impl`
- **Examples**:
  - ✅ `com.fcs.be.common.service.file.impl.FileServiceImpl`
  - ✅ `com.fcs.be.modules.iam.service.impl.JwtTokenServiceImpl`
  - ❌ `com.fcs.be.common.service.file.implements.FileServiceImpl` (DEPRECATED - Reserved keyword)

### Interface Packages
- **Pattern**: `com.fcs.be.<module>.service.interfaces`
- **Examples**:
  - `com.fcs.be.common.service.file.interfaces.FileService`
  - `com.fcs.be.modules.iam.service.interfaces.JwtTokenService`

### Test Implementation Packages
- **Pattern**: `com.fcs.be.<module>.service.impl` (same as implementation, different source root)
- **Location**: `src/test/java/com/fcs/be/<module>/service/impl/`
- **Examples**:
  - `src/test/java/com/fcs/be/common/service/file/impl/FileServiceImplTest.java`
  - `src/test/java/com/fcs/be/common/service/redis/impl/RedisServiceImplTest.java`

## Folder Structure Rules
```
src/main/java/com/fcs/be/
├── common/
│   └── service/
│       ├── file/
│       │   ├── interfaces/       (FileService.java)
│       │   └── impl/             (FileServiceImpl.java)
│       ├── redis/
│       │   ├── interfaces/       (RedisService.java)
│       │   └── impl/             (RedisServiceImpl.java)
│       └── notification/
│           ├── interfaces/       (NotificationService.java)
│           └── impl/             (NotificationServiceImpl.java)
└── modules/
    ├── iam/
    │   └── service/
    │       ├── interfaces/       (JwtTokenService.java)
    │       └── impl/             (JwtTokenServiceImpl.java)
    └── health/
        └── service/
            ├── interfaces/       (HealthService.java)
            └── impl/             (HealthServiceImpl.java)
```

## Deprecated
- ❌ `implements` package names - DO NOT USE
- All existing files have been migrated to `impl` naming convention
